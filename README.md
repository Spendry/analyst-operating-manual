# Analyst Operating Manual

> Part of my [portfolio](https://github.com/Spendry/portfolio).

An operating manual for someone who is not an analyst to sit in a junior quantitative research analyst's chair and learn the job by doing it. It defines the role, the minimal toolkit, the seven-step operating loop, the working standards, the daily ritual from power-on to shipped deliverable, and the rubric every project is reviewed against.

It covers two dialects of the same loop — Lean operations and quantitative finance — because once the loop is muscle memory, switching domains is switching vocabulary.

**Read it here: [analyst-operating-manual.md](analyst-operating-manual.md)**

## What's inside

- **The job**, defined in one sentence, and the three things a client pays for, in order
- **The kit**: Python + pandas, JupyterLab, matplotlib, git — and the rule for when a tool earns its way in
- **The seven-step loop**: Question, Pull, Profile, Clean, Explore, Answer, Communicate
- **Working standards** and notebook hygiene that make the work reviewable and reproducible
- **The project format**, time budgets, and the five-dimension review rubric

The manual references companion example notebooks and a deliverable reference; those live beside it in the working practice folder and will be added as the practice library grows.
